# 面试讲述稿

## 1 分钟版本

我把自己的无人机目标检测项目包装成了一个巡检 CV Agent。原始 CV 项目负责训练和推理，输出 bbox、类别和置信度；这个 Agent 项目负责把检测结果变成业务可用的异常事件、风险评分和巡检报告。

这样设计的好处是模型层和业务层解耦。后续无论换 YOLOv8、RT-DETR 还是 GroundingDINO，只要输出 schema 不变，Agent 的事件归并、风险评估和报告 API 都不用改。

## 面试追问

### 如何从检测结果变成 Agent？

我会把检测模型封装成 tool，输入是图片/视频/抽帧目录，输出是标准化 detection schema。Agent 再调用事件聚合、风险规则、报告生成工具，形成完整巡检流程。

### 如何降低误报？

第一层是置信度和类别过滤，第二层是多帧聚合，第三层结合风险区域和业务白名单，第四层把高风险事件交给人工复核。生产中还可以加入跟踪算法和历史轨迹对比。

### 如何部署？

轻量版本可以 FastAPI + Docker；真实推理需要 GPU 节点、模型权重管理、视频抽帧服务、异步任务队列和结果存储。前端或告警系统只调用 API，不直接接触模型。
